System Features

Privacy:

All information a user collects from a device is private and is not visible to any other registered user.

Installation:

To install on a private server, follow the instructions below
a. Download git on your machine
b. mkdir /var/www
c. cd /var/www
d. git clone https://github.com/kwyoung11/WaterLog.git /var/www/EnviroHub
e. cd EnviroHub
f. chmod +x installation_script.sh
g. ./installation_script.sh
This creates an administrator with credentials: email: admin@admin.com, password: admin

Public get api:

A user can select different features such as a certain data_type, time period, device and/or parameters when on EnviroHub.com. This sends an HTTP GET request to the server which then retrieves the data from the database and returns it to be displayed.

Security in Wireless Devices

The first part of the solution is to use digital signatures to ensure integrity and authenticity of the request. Each message is “signed” with the device’s private key given to it by EnviroHub.com. Signing the message ensures non-repudiation meaning that the server knows where the message is coming from and there is no question who is sending the message; only the server and the particular device know the shared key.
The second part of the solution is to use a symmetric shared secret key. There will be a unique secret key for every device that a user owns and each message will be encrypted on one side and decrypted on the other side using the same shared private key.

Password Security

All user passwords will be encrypted using a one-way hash function using a randomly generated salt. The number of login attempts will be limited to 3 before the user will be locked out of their account for 15 minutes.
