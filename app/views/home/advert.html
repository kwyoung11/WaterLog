EnviroHub

A Low Cost Environmental Monitoring Toolkit Source Code

Project

EnviroHub is a low-cost environmental monitoring toolkit that allows users to accurately gather and view a variety of environmental indicators. Similar systems can cost thousands of dollars [1], which was the motivation for this product. The software was designed with a public API in order to enable users to create and integrate their own environmental data devices. All of the code is open source to encourage expansion and improvement of the product. EnviroHub makes it simple and straightforward for users to see registered device locations on our website. Users can click on their device on a map and see all recent data recorded by that device. Data from any specific device is visualized via a graph so that users can see changes and trends over time and identify any issues with local environmental quality metrics. Data charts can be filtered by start and end dates and display a log of the latest recordings from the device. The currently developed hardware was designed and assembled with affordable and publicly accessible parts. The device created in this course was Arduino-centric.

Major features of the software include a wireless POST api endpoint for devices to send data wirelessly to, a GET API for query the server for data and devices, HTTPS support, user and device management, the ability to specify custom parameters for your device, the ability to compare device data to integrated data from sources such as the USGS and weather services, the ability to easily see trends in data charted across time, as well as privacy and security.

An EnviroHub web server can be easily installed on Debian/linux machines via an installation script. We thought that it was important to allow users to install our system on their own machines for privacy and security reasons. We know that data security and integrity is important for our users.

Hardware

The device created in this course cost $275 with a $6/month mobile data plan, and is capable of monitoring pH and temperature of waterways, and submitting the recorded data wirelessly to the server. The system is designed to be extendable, so that other Arduino compatible sensors can be purchased and integrated. With user and device registration on the EnviroHub website, Envirohub will then be capable of collecting and displaying data from the recordings wirelessly sent by the device. We decided to design the device in a light-weight fashion: instead of driving a fence post into the streambed and securing a PVC pipe to it that would house our sensors and datalogger, we decided to go with a more light-weight, easily maintainable route. With this floating, minimal design, you could easily move the device from location to location wihout much hassle.
